# 提示词库使用指南

> 从 9 个精选 GitHub 仓库同步优质提示词，一键注入到生成节点
> 路径：`👑 Tikpan 官方独家节点 / 05 提示词与分析 Prompt & Analysis / 提示词库 Prompt Library`

---

## 快速上手

### 第一步：同步提示词库（可选启用中文翻译）

在 ComfyUI 中添加 **"提示词库管理器"** 节点：

```
操作:           同步全部仓库
同步单个仓库:    全部
Tikpan_API密钥:  sk-xxxxxxxx   ← 选填,填了就把英文标题/预览翻译成中文
翻译模型:        deepseek-v4-flash
```

点击执行：
- **不填密钥**：10-30 秒抓完，下拉框显示英文
- **填密钥**：额外 1-2 分钟翻译时间，下拉框显示中文（**强烈推荐，500 张总成本不到 1 毛钱**）

**同步完成后会显示：**
```
📚 提示词库同步完成
总卡片数: 503
翻译条数: 1006 (模型: deepseek-v4-flash)
```

> 💡 翻译结果永久缓存在本地 JSON,以后打开 ComfyUI 不会再调 API。下次重新同步时会增量翻译,只翻新增/变更的卡片。

### 第二步：选择提示词

添加 **"提示词选择器"** 节点，下拉框会显示「中文标题 — 中文预览」格式：

```
选择提示词: 1. 电影感粉彩工作室人像 — 高分辨率柔和电影感人像,年轻女性...
过滤_标签: gpt-image-2
```

将 **"提示词文本"** 输出连接到你的图像/视频生成节点的 prompt 输入。

> ⚠️ **重要**：输出给生图节点的 `提示词文本` 仍然是**英文原文**——这样保证 GPT-Image-2、Nano Banana 等模型的生成质量。中文只用于下拉框可读性。

**完整工作流示例：**
```
[提示词选择器] → 提示词文本(英文) → [GPT Image 2 生图] → [保存图像]
```

## 工作原理

### 数据来源

提示词库从 9 个精选 GitHub 仓库自动抓取：

| 仓库 | 用途 |
|------|------|
| awesome-nano-banana-pro-prompts | Nano Banana 图像提示词 |
| awesome-gpt-image-2-API-and-Prompts | GPT-Image-2 图像提示词 |
| awesome-seedance-2.0-prompts | Seedance 视频提示词 |
| ai-mixed-cut | AI 混剪工作流 |
| ai-highlight-clip | 高光切片工作流 |
| ai-ttv-workflow | 文生视频流程 |
| video-mover | 视频搬运/适配 |
| BigBanana-AI-Director | AI 导演/多智能体 |

### 解析逻辑（v2 改进版）

1. 从 GitHub 获取仓库的 README.md
2. 按 `## H2 标题` 切分成多个段落
3. **章节黑名单过滤**（中英文双语）：跳过 Introduction / News / 快速开始 / 我的其他开源项目 / Acknowledge / License 等非提示词章节
4. **内容启发式过滤**：跳过 `npx`、`pip install`、`git clone`、`Welcome to` 等明显是文档/安装指引的段落
5. **H2 必须包含代码块或 blockquote**：纯软件 README（如 toki-plus 系列）自动 0 卡片
6. **按 H3 拆细颗粒度**：一个 H2 分类下若有多个 H3 用例（如 GPT-Image-2 的 100+ 个 Case），每个 H3 独立成卡
7. **优先识别 `#### 📝 Prompt` 子节**：Nano Banana 仓库的真实提示词被精准提取
8. 每张卡片提取：
   - **标题**：清理 emoji 和 markdown 链接语法
   - **提示词内容**：优先代码块 → blockquote → 长段落
   - **标签**：自动识别 `Tags: xxx` 或从仓库继承
9. **可选翻译**：填了 API 密钥则调 `deepseek-v4-flash` 把标题和 80 字预览译成中文
10. 保存到本地 `data/prompts-library/` 目录（原子写入，断电不损坏）

### 本地存储

同步后的数据保存在：
```
data/prompts-library/
├── repo_YouMind-OpenLab__awesome-nano-banana-pro-prompts.json
├── repo_EvoLinkAI__awesome-gpt-image-2-API-and-Prompts.json
└── ...
```

**离线使用**：同步后可断网使用，数据永久保存在本地。

## 三个节点详解

### 1. 提示词库管理器

**用途**：同步、管理、翻译提示词库

**输入参数：**
- `操作`：查看状态 / 同步全部仓库
- `同步单个仓库`：选择特定仓库单独同步
- `Tikpan_API密钥`（选填）：填写后会把英文标题和 prompt 预览翻译成中文存入 JSON
- `翻译模型`：`deepseek-v4-flash`（高性价比，500 张全译 ≈ ¥0.05）

**输出：**
- `状态报告`：同步结果、卡片数、翻译条数

**断点续传机制：**
- 每 10 条翻译后立即写入磁盘（原子写入）
- 中途 ComfyUI 崩溃或网络断开 → 已翻译部分永久保留
- 下次同步自动跳过已译卡片，只翻译剩余部分
- 最坏情况浪费 = 1 个批次 = 不到 1 分钱

**使用场景：**
- 首次使用时同步
- 每周更新一次（仓库会持续更新，增量翻译几乎免费）

---

### 2. 提示词选择器（3 个变体）

**用途**：从下拉列表选择提示词。根据你的工作流类型选不同变体：

| 节点 | 下拉显示 | 适用场景 |
|------|---------|---------|
| **提示词选择器·全部** | 全部 503 张 | 同时做图片+视频，不在乎类型 |
| **提示词选择器·图片** | 仅图片类 336 张 | GPT-Image-2 / Nano Banana 等生图工作流 |
| **提示词选择器·视频** | 仅视频类 163 张 | Seedance 等生视频工作流 |

**输入参数（3 个变体相同）：**
- `选择提示词`：下拉列表（显示前 800 个，标题+30 字预览，优先中文）
- `过滤_仓库`：进一步限定仓库（专用版只列含本类卡片的仓库）
- `过滤_标签`：按标签筛选（专用版自动剔除冗余标签）
- `搜索关键词`：输入关键词搜索（下拉为占位符时生效）
- `显示详情`：是否在控制台显示完整信息（含中英对照）

**输出：**
- `提示词文本`：**英文原文** prompt（连接到生成节点）
- `卡片标题`：提示词标题
- `卡片详情`：包含中文预览、来源、标签、链接、英文原文

**适合场景**：浏览和选择提示词。**推荐用专用版**——下拉框更短、不会误用错类型。

---

### 3. 提示词搜索

**用途**：高级搜索功能

**输入参数：**
- `搜索关键词`：输入关键词（必填）
- `过滤_仓库`：按仓库过滤
- `过滤_标签`：按标签过滤
- `最多返回数量`：限制结果数量（1-50）
- `结果索引`：选择第几个结果（0 = 第一个）

**输出：**
- `提示词文本`：选中的提示词内容
- `卡片标题`：选中的提示词标题
- `搜索结果列表`：所有匹配结果的列表
- `结果总数`：匹配的总数量

**适合场景**：快速查找特定类型的提示词

## 实用技巧

### 按模型类型过滤

不同模型适合不同提示词：

```
GPT-Image-2  → 过滤_标签: gpt-image-2
Nano Banana  → 过滤_标签: nano-banana
视频生成      → 过滤_标签: video 或 seedance
```

### 组合过滤

同时使用多个条件缩小范围：

```
过滤_仓库: EvoLinkAI/awesome-gpt-image-2-API-and-Prompts
过滤_标签: image
搜索关键词: 商品
```

### 自定义提示词库

在 `data/prompts-library/` 创建自己的 JSON 文件：

```json
{
  "slug": "my-prompts",
  "updatedAt": "2026-05-31T10:00:00Z",
  "cards": [
    {
      "id": "my-1",
      "repo": "my-prompts",
      "title": "我的提示词",
      "prompt": "这里是提示词内容...",
      "tags": ["custom"],
      "body": "",
      "url": ""
    }
  ]
}
```

## 常见问题

**Q: 同步失败怎么办？**

检查网络连接，稍后重试。可以尝试同步单个仓库定位问题。

**Q: 提示词列表为空？**

需要先使用"提示词库管理器"同步，然后**重启 ComfyUI 或 F5 刷新浏览器**——ComfyUI 的下拉列表在节点首次加载时就锁定了，需要让前端重新读取。

**Q: 同步成功了但下拉框还是英文？**

填了 API 密钥才会翻译。如果填了密钥还是英文：
1. 看控制台有没有 `[翻译进度]` 输出
2. 看状态报告里 `翻译条数` 是不是 > 0
3. 重启 ComfyUI 后再次拖入「提示词选择器」节点（旧节点保留的是同步前的下拉缓存）

**Q: 翻译到一半 ComfyUI 崩了，重新同步会重复花钱吗？**

不会。每 10 条翻译就写盘一次，下次同步自动跳过已译部分，最多浪费 1 个批次（10 条 ≈ 1 分钱）。

**Q: 我不想花钱翻译，能用英文版吗？**

把 `Tikpan_API密钥` 字段留空即可，下拉框显示英文，所有功能照常工作。

**Q: 提示词太多，如何快速找到想要的？**

使用"提示词搜索"节点，输入具体关键词（如 "商品"、"人物"、"product"、"portrait"）。中英文都能搜——搜索会同时匹配中文译文和英文原文。

**Q: 输出给生图节点的 prompt 是中文还是英文？**

**始终是英文原文**。GPT-Image-2、Nano Banana 等模型对英文 prompt 训练更充分，中文 prompt 会大幅降低生成质量。中文译文仅用于下拉框可读性，不影响实际生成。

**Q: 为什么有的仓库同步出 0 张卡片？**

`toki-plus/*` 几个仓库是软件项目 README（不含真实提示词），新解析器会自动识别并跳过。这是预期行为。

**Q: 如何查看提示词完整内容？**

勾选"显示详情"，执行后查看控制台输出。控制台会同时显示中文预览和英文原文。

```
[提示词搜索]

  过滤_标签: gpt-image-2
  ↓
[GPT Image 2 官方生图]
  生成指令: (连接提示词文本)
  ↓
[保存图像]
```

```
[提示词选择器]
  选择提示词: 15. 产品展示视频

[Veo 3.1 视频生成]
  提示词: (连接提示词文本)
  ↓
```
