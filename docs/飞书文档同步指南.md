# 飞书文档同步指南

本项目支持将本地 Markdown 文档自动同步到飞书云文档，方便团队协作和在线查看。

## 功能说明

`scripts/sync_to_feishu.py` 是一个轻量级的飞书文档同步工具，可以：

- 将本地 Markdown 文件转换为飞书文档格式（标题、代码块、列表等）
- 自动清空旧内容并写入新内容（完全覆盖模式）
- 支持批量同步或单文件同步
- 自动管理 token 缓存（2小时有效期）

## 配置步骤

### 1. 创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 创建企业自建应用
3. 在"凭证与基础信息"页面获取：
   - `App ID`
   - `App Secret`
4. 在"权限管理"中开通以下权限：
   - `docx:document` - 查看、评论、编辑和管理云文档
   - `docx:document:readonly` - 查看云文档

### 2. 准备飞书文档

1. 在飞书云文档中创建需要同步的文档
2. 从文档 URL 中提取 `document_id`（`/docx/` 后面那段）
   - 例如：`https://example.feishu.cn/docx/AbCdEfGhIjKl` 
   - `document_id` 就是 `AbCdEfGhIjKl`
3. 将文档权限设置为"应用可编辑"

### 3. 配置同步脚本

```bash
# 复制配置模板
cp .feishu_config.example.json .feishu_config.json

# 编辑配置文件
# Windows: notepad .feishu_config.json
# Mac/Linux: nano .feishu_config.json
```

配置文件示例：

```json
{
  "app_id": "cli_a1b2c3d4e5f6g7h8",
  "app_secret": "your_app_secret_here_32_chars",
  "files": {
    "docs/节点使用教程.md": "AbCdEfGhIjKl1234567890",
    "docs/节点速查表.md": "XyZaBcDeFgHi0987654321",
    "CHANGELOG.md": "MnOpQrStUvWx1122334455"
  }
}
```

**字段说明**：
- `app_id`：飞书应用的 App ID
- `app_secret`：飞书应用的 App Secret（**敏感信息，不要提交到 git**）
- `files`：本地文件路径 → 飞书文档 ID 的映射表

## 使用方法

### 同步所有配置的文档

```bash
python scripts/sync_to_feishu.py
```

输出示例：

```
📄 同步: 节点使用教程.md -> docx:AbCdEfGh...
  🧹 清空旧内容: 45 blocks
  ✅ 完成

📄 同步: 节点速查表.md -> docx:XyZaBcDe...
  🧹 清空旧内容: 23 blocks
  ✅ 完成

🎉 同步完成
```

### 仅同步指定文档

```bash
python scripts/sync_to_feishu.py docs/节点使用教程.md
```

### 在 Git 提交后自动同步（可选）

创建 Git Hook `.git/hooks/post-commit`：

```bash
#!/bin/bash
# 检查是否修改了文档文件
if git diff --name-only HEAD~1 HEAD | grep -E "^docs/.*\.md$|^CHANGELOG\.md$"; then
    echo "检测到文档更新，同步到飞书..."
    python scripts/sync_to_feishu.py
fi
```

## 支持的 Markdown 语法

当前版本支持以下 Markdown 元素的转换：

| Markdown | 飞书文档 | 示例 |
| --- | --- | --- |
| `# 标题` | 一级标题 | `# 章节标题` |
| `## 标题` | 二级标题 | `## 小节标题` |
| `### 标题` | 三级标题 | `### 子标题` |
| `#### 标题` | 四级标题 | `#### 细分标题` |
| `` ```代码块``` `` | 代码块 | 支持语言高亮 |
| `- 列表项` | 无序列表 | `- 第一项` |
| `* 列表项` | 无序列表 | `* 第二项` |
| 普通文本 | 文本块 | 段落内容 |

**暂不支持**（会转为普通文本）：
- 表格（`| 列1 | 列2 |`）
- 图片（`![alt](url)`）
- 链接（`[text](url)`）
- 粗体/斜体等行内格式

## 常见问题

### Q1: 提示 "获取 tenant_access_token 失败"

**原因**：App ID 或 App Secret 填写错误。

**解决**：
1. 检查 `.feishu_config.json` 中的凭证是否正确
2. 确认应用状态为"已启用"
3. 重新复制粘贴凭证，避免多余空格

### Q2: 提示 "读取文档 blocks 失败"

**原因**：应用没有文档权限或 document_id 错误。

**解决**：
1. 在飞书开放平台检查应用权限是否开通
2. 确认 document_id 是否正确（从 URL 中提取）
3. 确认文档权限设置为"应用可编辑"

### Q3: 同步后文档格式不对

**原因**：Markdown 语法不在支持列表中。

**解决**：
1. 检查是否使用了表格、图片等暂不支持的语法
2. 将复杂格式改为纯文本或代码块
3. 如需扩展支持，可修改 `_markdown_to_blocks()` 方法

### Q4: 同步速度慢

**原因**：飞书 API 限制单次最多 40 个 block，大文档需要分批写入。

**解决**：
- 正常现象，每批次间隔 0.3 秒
- 如果文档超过 1000 行，建议拆分为多个文档

### Q5: 误删了飞书文档内容怎么办？

**原因**：同步脚本是完全覆盖模式，会清空旧内容。

**解决**：
1. 飞书文档有版本历史，可以在文档右上角"更多 → 版本历史"中恢复
2. 本地 Markdown 文件是源文件，可以重新同步

## 安全提示

⚠️ **重要**：`.feishu_config.json` 包含敏感凭证，已加入 `.gitignore`。

**请勿**：
- 将 `.feishu_config.json` 提交到 git
- 在公开场合分享 App Secret
- 将配置文件上传到公开服务器

**建议**：
- 定期轮换 App Secret
- 仅在可信环境运行同步脚本
- 团队协作时每人使用独立的飞书应用

## 技术细节

### API 端点

- 获取 token：`POST /open-apis/auth/v3/tenant_access_token/internal`
- 读取文档：`GET /open-apis/docx/v1/documents/{document_id}/blocks`
- 删除内容：`DELETE /open-apis/docx/v1/documents/{document_id}/blocks/{block_id}/children/batch_delete`
- 写入内容：`POST /open-apis/docx/v1/documents/{document_id}/blocks/{block_id}/children`

### Token 缓存

- Token 有效期：2 小时
- 自动刷新：过期前 5 分钟自动重新获取
- 缓存位置：内存中（不写入文件）

### 批量写入

- 单次限制：40 个 block
- 分批策略：自动分批，每批间隔 0.3 秒
- 失败重试：当前版本不自动重试，失败会抛出异常

## 扩展开发

如需支持更多 Markdown 语法，可修改 `FeishuClient._markdown_to_blocks()` 方法：

```python
# 示例：添加表格支持
if line.startswith("|"):
    # 解析表格行
    cells = [cell.strip() for cell in line.split("|")[1:-1]]
    blocks.append({
        "block_type": 15,  # 表格类型
        "table": {
            "cells": cells,
            # ... 更多表格配置
        }
    })
```

飞书文档 block 类型参考：
- 2: 文本
- 3: 一级标题
- 4: 二级标题
- 5: 三级标题
- 6: 四级标题
- 12: 无序列表
- 14: 代码块
- 15: 表格

完整 API 文档：https://open.feishu.cn/document/server-docs/docs/docs/docx-v1/document/overview

## 维护建议

1. **定期检查权限**：飞书应用权限可能过期，建议每季度检查一次
2. **备份配置**：将 `.feishu_config.json` 备份到安全位置（不要提交到 git）
3. **监控同步日志**：如果同步失败，检查控制台输出的错误信息
4. **版本控制**：本地 Markdown 文件始终是源文件，飞书文档仅作为展示
