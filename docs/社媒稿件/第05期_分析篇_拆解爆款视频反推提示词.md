### 为什么要用 AI 分析视频？

很多做内容的朋友，看到一条爆款视频会模仿，但模仿出来效果就是差一截。原因是"看上去像，但底层逻辑没学到"。

AI 视频分析做的事情，是把你凭直觉感受到的东西，变成可以量化、可以复用的结构化信息。

今天这期，教你用攀升AI的分析节点，把一条爆款视频拆解成可以直接拿来生成新内容的提示词。

`配图：原爆款视频截帧 + AI分析报告文本截图，对比展示`

### 核心节点介绍

**① Gemini 3 Flash Preview 图片/视频分析（`TikpanGemini3FlashPreviewAnalystNode`）**

能分析：
- 图片（最多4张直接接入，或图片URL列表）
- 视频（视频URL、本地小视频、视频帧抽帧）

输出：
- 分析报告（文字）
- 反推提示词（直接可用）
- 结构化 JSON（适合后续处理）
- token 用量（便于计费）

`配图：Gemini 3 Flash 节点参数完整截图`

**② GPT-5.4 Mini 多模态推理（`TikpanGPT5MiniResponsesNode`）**

更适合：
- 商品卖点提炼
- 广告文案优化
- 文件（PDF/Excel）分析
- 联网搜索增强分析

**③ Grok 多图剧本重构（`TikpanGrokPromptOptimizerNode`）**

专门把分析报告转化成适合 Grok / Veo / HappyHorse 视频节点使用的提示词。

---

### 视频拆解工作流

**方式一：直接传视频URL（推荐）**

```
[视频URL 文本输入] → [Gemini 3 Flash 分析: 视频URL输入]
[Gemini 3 Flash 分析] → [分析报告文本输出]
```

参数设置：
- 分析任务：`视频分镜拆解`
- 输出格式：`Markdown结构化`
- 视频输入策略：`自动优先`

`配图：填写视频URL的节点截图`

**方式二：视频帧抽帧分析**

当你有本地视频但无法获取公开URL时：

```
[LoadVideo → 视频帧 IMAGE] → [Gemini 3 Flash 分析: 视频帧_IMAGE输入]
[Gemini 3 Flash 分析] → [分析报告]
```

抽帧策略建议选`混合智能`：兼顾均匀覆盖和画面变化，适合大多数视频类型。

`配图：抽帧工作流截图`

### 分析报告解读

AI 输出的分析报告，通常包含：

**镜头层面：**
- 每段时间的镜头内容描述
- 运镜方式（推进/环绕/拉远/固定）
- 景别变化（远景→中景→特写节奏）

**视觉层面：**
- 主色调特征
- 灯光风格
- 画面构图方式

**叙事层面：**
- 情绪节奏变化
- 商品/主体出现时机
- 钩子设计分析

**提示词建议：**
- 直接可以复制到视频生成节点使用的文本

`配图：完整分析报告截图示例，标注各个维度的内容区域`

---

### 从分析到生成：完整链路

```
[分析目标视频]
      ↓
[Gemini 3 Flash 分析 → 分析报告]
      ↓
[Grok 多图剧本重构]
  ↳ 输入：Gemini原片拆解报告 + 你的产品描述 + 运镜微调要求
  ↳ 输出：Grok 专属提示词
      ↓
[视频生成节点（HappyHorse/Veo/Vidu3）]
      ↓
[生成同款风格视频]
```

`配图：完整工作流图，4个节点串联的截图`

Grok 重构节点的输入建议这样写：
- **核心产品与植入场景**：你的产品名称 + 希望它怎么出现
- **氛围与运镜微调**：需要调整的情绪节奏、运镜方式差异

- 适合的目标受众描述
- 推荐的内容角度和话题切入点
- 可以直接用于图文/视频的文案参考

### Gemini 3.5 Flash：处理长文档

如果你有竞品分析报告、行业白皮书、品牌手册等长 PDF 文件，
用 Gemini 3.5 Flash 节点可以直接分析：

- 填入本地文件路径
- 任务类型选`长文档/PDF总结`
- 开启 thinking 深度推理获得更细致的洞察

### 小结

这期核心链路：
