## 1. 快速开始

### 1.1 安装位置

把整个插件目录放到 ComfyUI 的自定义节点目录：

```text
ComfyUI/custom_nodes/ComfyUI-Tikpan-Pro
```

然后重启 ComfyUI。右键画布添加节点时，找到：

```text
👑 Tikpan 官方独家节点
```

### 1.2 获取 API Key

1. 打开 https://tikpan.com 并登录账号。
2. 进入控制台或个人中心，创建 API Key。
3. 在节点的 `api_key`、`API_密钥` 或 `Tikpan_API密钥` 字段中填入密钥。

不同节点的密钥字段名称略有不同，但含义一致：都是 Tikpan 的接口调用凭证。

### 1.3 最小可用工作流

文生图：

```text
[Tikpan: GPT-Image-2 官方正式版(生图)] -> [PreviewImage]
```

图生图或参考图生成：

```text
[LoadImage] -> [参考图/主图输入]
[Tikpan: Gemini 14图极限生图] -> [PreviewImage]
```

文生视频：

```text
[Tikpan: HappyHorse 1.0 T2V 文生视频] -> [PreviewVideo 或视频保存节点]
```

音乐生成：

```text
[Tikpan: Suno 音乐生成] -> [PreviewAudio 或 SaveAudio]
```

语音合成：

```text
[Tikpan: speech-2.8-hd 高清语音合成] -> [PreviewAudio 或 SaveAudio]
[Tikpan: speech-2.8-turbo 极速语音合成] -> [PreviewAudio 或 SaveAudio]
[Tikpan: Gemini 3.1 Flash TTS Preview] -> [PreviewAudio 或 SaveAudio]
```

## 3. 通用参数怎么理解

| 参数 | 含义 | 建议 |
| --- | --- | --- |
| `api_key` / `API_密钥` / `Tikpan_API密钥` | Tikpan 调用凭证 | 不要截图公开，不要提交到公开仓库 |
| `prompt` / `生成指令` / `修改指令` | 你要模型完成的任务 | 写清主体、场景、风格、构图、限制 |
| `negative_prompt` / `负面风格标签` | 不希望出现的内容 | 用于排除低质量、错误风格或不需要的元素 |
| `seed` | 随机种子 | 固定 seed 方便复现；随机 seed 方便探索 |
| `resolution` / `分辨率档位` | 输出清晰度 | 草稿用低档，最终图再升高 |
| `ratio` / `aspect_ratio` / `画面比例` | 横竖屏比例 | TikTok/短视频常用 `9:16`，横版常用 `16:9` |
| `duration` | 视频时长 | 时长越长越慢，费用通常也更高 |
| `watermark` | 是否带水印 | 商用素材通常关闭水印 |
| `mode` | 同步或异步提交 | 长视频建议异步，再用 Task Fetcher 查询 |
| `max_wait_seconds` | 最长等待时间 | 同步等待时使用；过短会提前超时 |
| `poll_interval` | 查询间隔 | 一般 5-10 秒即可，不建议太频繁 |
| `代理端口` | 本地代理端口 | 只有部分旧链路需要，常见为 `10808` 或 `7890` |

## 4. 输出怎么连接

### 4.1 图片输出

图片节点一般输出：

```text
IMAGE
STRING 日志
```

连接方式：

```text
[Tikpan 图片节点: IMAGE] -> [PreviewImage]
[Tikpan 图片节点: IMAGE] -> [SaveImage]
```

日志字符串可以接到文本显示节点，也可以直接看 ComfyUI 控制台。日志里通常包含请求参数、下载路径、错误信息和上游返回摘要。

### 4.2 视频输出

视频节点一般输出：

```text
本地保存路径
任务ID
云端链接
完整日志
VIDEO
```

推荐同时保留 `任务ID` 和 `云端链接`。如果 ComfyUI 当前环境没有视频预览节点，`VIDEO` 不能直接看也没关系，本地保存路径和云端链接仍然可用。

### 4.3 音频输出

Suno 节点会输出：

```text
音频路径1 / 音频路径2
音频链接1 / 音频链接2
任务ID
片段ID
AUDIO 1
AUDIO 2
```

连接方式：

```text
[Tikpan: Suno 音乐生成: AUDIO] -> [PreviewAudio]
[Tikpan: Suno 音乐生成: AUDIO] -> [SaveAudio]
```

speech-2.8-hd / speech-2.8-turbo 节点会输出单路语音：

```text
音频路径
音频链接
任务ID
文件ID
计费字符数
状态日志
AUDIO
```

连接方式：

```text
[Tikpan: speech-2.8-hd 高清语音合成: AUDIO] -> [PreviewAudio]
[Tikpan: speech-2.8-turbo 极速语音合成: AUDIO] -> [PreviewAudio]
```

Gemini TTS 节点会输出单路 WAV 音频，并额外输出接口路径和 token 用量：

```text
音频路径
音频链接
输入输出用量
接口路径
状态日志
AUDIO
```

连接方式：

```text
[Tikpan: Gemini 3.1 Flash TTS Preview: AUDIO] -> [PreviewAudio]
[Tikpan: Gemini 3.1 Flash TTS Preview: AUDIO] -> [SaveAudio]
```

## 5. 图片节点使用方法

### 5.1 GPT-Image-2 官方正式版(生图)

推荐用于稳定的商业图像生成。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | 填 Tikpan API Key |
| `生成指令` | 写完整画面要求 |
| `模型` | 默认 `gpt-image-2` |
| `分辨率档位` | `512`、`1K`、`2K`、`4K` 等 |
| `画面比例` | `1:1`、`16:9`、`9:16`、`4:3`、`3:4`、`21:9`、`9:21` |
| `画质与推理强度` | `auto`、`low`、`medium`、`high` |
| `审核强度` | `auto`、`low`、`high` |
| `返回格式` | `png`、`webp`、`jpg` |
| `跳过错误` | 批量工作流建议开启，失败时不中断整个流程 |

推荐工作流：

```text
[Tikpan: GPT-Image-2 官方正式版(生图)] -> [PreviewImage]
                                             -> [SaveImage]
```

提示词示例：

```text
一张竖屏 9:16 的商业产品海报，主体是一瓶透明香水，放在潮湿的黑色岩石上，背景是柔和的晨光和浅雾，真实摄影风格，高级杂志广告质感，清晰产品标签，画面干净，不要多余文字。
```

### 5.2 GPT-Image-2 官方正式版(修图) V2

推荐用于编辑已有图片，支持主图、参考图、遮罩、生成多张。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `主图像` | 必填，要编辑的原图 |
| `编辑指令` | 说明要修改什么、保留什么 |
| `生成张数` | 一次返回多张变体 |
| `分辨率档位` | `Auto` 会尽量保留原图尺寸逻辑 |
| `画面比例` | `Auto` 会沿用原图比例 |
| `背景模式` | `auto`、`opaque`、`transparent` |
| `遮罩反相` | 遮罩方向不对时切换 |
| `提示增强` | 建议开启，提高稳定性 |
| `参考图1-4` | 放产品、人物、风格参考 |
| `参考流` | 可接上一节点的图片批次 |
| `遮罩掩码` | 精准控制编辑区域 |

无遮罩修图：

```text
[LoadImage: 原图] -> [主图像]
[Tikpan: GPT-Image-2 官方正式版(修图) V2] -> [PreviewImage]
```

遮罩修图：

```text
[LoadImage: 原图] -> [主图像]
[MaskEditor/遮罩节点] -> [遮罩掩码]
[Tikpan: GPT-Image-2 官方正式版(修图) V2] -> [PreviewImage]
```

遮罩规则通常是：白色区域表示要修改，黑色区域表示保留。如果结果刚好反了，打开 `遮罩反相`。

### 5.3 GPT-Image-2-all 视觉建筑师(生成)

适合多参考图生成，最多可接 14 张参考图。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | Tikpan API Key |
| `生成指令` | 说明如何使用参考图 |
| `分辨率档位` | `512`、`1K`、`2K`、`4K` |
| `画面比例` | 常用 `1:1`、`9:16`、`16:9` |
| `品质` | `standard` 更快，`hd` 更精细 |
| `参考图_1~14` | 放人物、产品、风格、场景参考 |

工作流：

```text
[LoadImage: 产品] -> [参考图_1]
[LoadImage: 风格] -> [参考图_2]
[Tikpan: GPT-Image-2-all 视觉建筑师(生成)] -> [PreviewImage]
```

### 5.4 GPT-Image-2-all 视觉整形师(修改)

适合在已有底图上做局部重绘、换产品、换背景。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `底图` | 必填，要修改的图片 |
| `修改指令` | 写清修改区域和保留区域 |
| `遮罩_Mask` | 可选，白色通常代表修改区域 |
| `产品参考图` | 可选，要植入或保持一致的产品 |
| `品质` | `standard` 或 `hd` |

工作流：

```text
[LoadImage: 底图] -> [底图]
[LoadImage: 产品] -> [产品参考图]
[MaskEditor] -> [遮罩_Mask]
[Tikpan: GPT-Image-2-all 视觉整形师(修改)] -> [PreviewImage]
```

### 5.5 Gemini 14图极限生图

适合多图融合、角色一致性、风格迁移，最多可接 14 张参考图。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | Tikpan API Key |
| `修改指令` | 既可文生图，也可要求参考图融合 |
| `模型` | 根据节点下拉选择 |
| `分辨率` | `1K`、`2K`、`4K` 等 |
| `画面比例` | 选择目标比例 |
| `调用方式` | 按节点提供的链路选择 |
| `参考图_1~14` | 多图输入 |

提示词建议：

```text
请参考图1的人物脸部特征，参考图2的服装材质，参考图3的拍摄光线，生成一张竖屏商业短视频封面。保持人物身份一致，背景更换为高端商场橱窗，真实摄影风格。
```

### 5.6 Nano Banana Pro

适合 Gemini / Banana Pro 图像链路，支持多参考图、温度和 `max_tokens` 控制。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `调用方式` | 在节点中选择 Gemini 原生或兼容调用方式 |
| `模型` | 如 `gemini-3-pro-image-preview` 等 |
| `修改指令` | 生成或编辑要求 |
| `分辨率` | `1K`、`2K`、`4K` 或 `none` |
| `温度` | 越高越发散，越低越稳定 |
| `max_tokens` | 控制响应文本/多模态预算 |
| `参考图_1~14` | 可选 |

### 5.7 豆包图像生成 Seedream

适合使用 Doubao 图像链路，支持联网搜索增强、多图生成和多种输出格式。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `api_key` | Tikpan API Key |
| `prompt` | 生成提示词 |
| `size_mode` / `size_quality` | 控制尺寸和质量 |
| `aspect_ratio` | 画面比例 |
| `output_format` | 输出格式 |
| `watermark` | 是否加水印 |
| `联网搜索增强` | 需要结合实时信息时开启 |
| `多图生成` | 一次生成多张 |
| `最多生成张数` | 控制批量数量 |
| `input_image` | 可选参考图 |
| `image_url_or_base64` | 可传图片 URL 或 base64 |
| `negative_prompt` | 可选负面提示词 |

### 5.8 Grok Imagine Image 生图 / Pro

适合用 Grok 图像模型文生图。有两个档位：标准版（`grok-imagine-image`，约 0.208 元/张）和 Pro 版（`grok-imagine-image-pro`，约 0.728 元/张）。节点通过幂等键避免网络抖动时重复扣费。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | Tikpan API Key |
| `生成指令` | 生成描述，建议英文效果更稳 |
| `生成张数` | 一次返回多张（最多 10 张） |
| `画面比例` | 支持 `auto`、`1:1`、`16:9`、`9:16`、`4:3`、`3:4`、`3:2`、`2:3` 等 |
| `清晰度` | `auto`、`1k`、`2k` |
| `兼容尺寸` | 需要精确尺寸时填写，如 `1024x1024` |
| `跳过错误` | 批量工作流建议开启，失败时输出黑图而不中断 |

工作流：

```text
[Tikpan: Grok Imagine Image 生图] -> [PreviewImage]
[Tikpan: Grok Imagine Image Pro 生图] -> [SaveImage]
```

常用字段：

| 字段 | 用法 |
| --- | --- |

| `参考图2` / `参考图3` | 可选，追加参考 |
| `编辑指令` | 说明要做什么、保留什么 |
| `生成张数` | 一次返回多张 |
| `画面比例` / `清晰度` | 同生图节点 |

工作流：

```text
[LoadImage: 参考图] -> [参考图1]
[Tikpan: Grok Imagine Image 参考图/修图] -> [PreviewImage]
```

### 5.10 Qwen-Image-2.0 生图/编辑

适合通义千问图像模型，支持文生图、图生图/编辑、多参考图三种模式，最多传 4 张参考图。

常用字段：

## 9. 工具节点使用方法

### 9.1 API 多模型并发生图引擎

在单节点内管理多个 API 请求，适合模型对比、容灾备份和批量出图。三种策略：

| 策略 | 行为 | 适合场景 |
| --- | --- | --- |
| `failover` | 按顺序逐个尝试，成功即停 | 日常省钱，正式商用推荐 |
| `race_first_success` | 全部并发，谁先成功就返回 | 抢速度，可能多消耗余额 |
| `parallel_all` | 全部跑完，合并成功图片 batch | 模型对比、批量出图、压测 |

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_Key` | Tikpan API Key |
| `Prompt` | 生成提示词 |
| `Models` | 每行一个模型 ID，如 `grok-imagine-image` |
| `Strategy` | 选择上表三种策略之一 |
| `Max_Concurrency` | 最大并发数，`failover` 建议 1 |
| `Images_Per_Model` | 每个模型生成几张 |

输出：`Images`（图片 batch）、`Stage_Log`（链路日志）、`Result_JSON`（每次请求详情）。

工作流：

```text
[Tikpan: API 多模型并发生图引擎: Images] -> [PreviewImage]
```

### 9.2 异步图片任务组（Submit / Result / Join / List）

这套节点把 API 请求放进后台任务池，让多个图片任务可以先全部提交，后台并行执行，再统一收图。是最接近"无限画布"并行体验的工作流。

**Submit（提交）**：提交一个任务并立即返回 `Task_ID`，不等生成完成。

```text
[Tikpan: 异步提交图片任务] -> Task_ID
```

**Result（查询单任务）**：查询指定任务状态。

| 模式 | 行为 |
| --- | --- |
| `return_now` | 立即返回当前状态（不管是否完成） |
| `wait_until_done` | 等待到成功、失败或超时 |

**Join（合并多任务）**：接收最多 8 个 `Task_ID`，统一等待完成，把成功图片合并成一个 batch。这是推荐的多任务收图方式。

```text
[Submit 节点 1] -> Task_ID_1
[Submit 节点 2] -> Task_ID_2
[Submit 节点 3] -> Task_ID_3
                 -> [Tikpan: 合并异步图片任务: Images] -> [PreviewImage]
```

**List（最近任务列表）**：查看当前 ComfyUI 进程里的异步任务历史，方便调试。

任务图片和 JSON 会保存到 `ComfyUI/output/TikpanAsync/`。

| --- | --- |
| 一张复杂图自动识别产品/背景/文字并分层 | `工具｜智能分层 PSD 生成器` |
| 提前下载好分层节点的依赖和模型，避免首次卡顿 | `工具｜PSD 模型预下载器` |

### 10.2 智能分层 PSD 生成器 (`TikpanSmartPSDLayeringNode`)

把一张图自动拆解成多个图层，每个独立元素一层。**纯本地处理，不依赖任何 API**。

```text
[任意生图节点] -> [Tikpan: 智能分层 PSD 生成器] -> output/xxx.psd
                                              -> 预览图（图层布局）
                                              -> 分层日志
```

**三档分层精度**（按需选择，下载量与耗时不同）：

| 档位 | 模型 | 下载体积 | 单图耗时 | 效果 | 适合场景 |
| --- | --- | --- | --- | --- | --- |
| 经济档 | BiRefNet + OpenCV 连通域 + PaddleOCR | 按需下载 | 5-15秒 | ⭐⭐⭐ | 简单商品图、追求速度 |
| 标准档 ⭐推荐 | BiRefNet + SAM2 自动多尺度 + PaddleOCR | 按需下载 | 20-60秒 | ⭐⭐⭐⭐ | 大多数日常场景、复杂商品图、海报 |
| 极致档 | BiRefNet + GroundingDINO + SAM2 + LaMa | 按需下载 | 60-180秒 | ⭐⭐⭐⭐⭐ | 商业级、复杂海报、被遮挡区域智能补全 |

**关键参数**：
- `分层档位`：经济 / 标准 / 极致。第一次选哪档就只下哪档的模型，不会全下。
- `补全被遮挡区域`：标准档勾选这个 = 极致档效果。让每一层都是完整的（不是带洞的），适合后期商业修改。
- `检测文字`：海报、广告图建议开启，会优先用 PaddleOCR 把每段文字独立成层，失败时降级到 EasyOCR。
- `最小元素面积`：过滤小噪点（默认 2000 像素，复杂图可调到 5000）。
- `边缘羽化`：让分层边缘更自然（默认 5）。
- `自动安装依赖`：首次使用必须开启，否则报错。

**输出图层结构**（自下而上）：

```text
原图_参考       ← 默认隐藏，作为兜底
背景             ← 主体外的部分
元素_1（最大）   ← 按面积从大到小排序
元素_2
元素_3
...
文字_1_xxx       ← OCR 检测到的每段文字独立成层
文字_2_xxx
```

**首次使用提示**：

> ⏳ 第一次运行此节点时（按所选档位）会自动安装依赖并下载模型：
> - 经济档：BiRefNet、PaddleOCR、OpenCV，并保留 rembg 兜底。
> - 标准档：在经济档基础上启用 SAM2 自动多尺度分割。
> - 极致档：在标准档基础上启用 GroundingDINO 语义分割和 LaMa 补全。
> 
> 如果不想等待，可以先用下面介绍的"PSD 模型预下载器"提前下载。

### 10.3 PSD 模型预下载器 (`TikpanPSDDependencyDownloaderNode`)

让用户提前一次性下好分层节点需要的全部依赖和模型，避免首次使用卡顿。

```text
[Tikpan: PSD 模型预下载器（档位选择）] -> 下载日志
                                    -> 状态预览图
```

**用法**：拖到画布，选择档位（经济 / 标准 / 极致 / 全部），点击运行，等几分钟下完即可。后续再用智能分层节点就秒开。

**关键参数**：
- `下载档位`：建议选"标准档"或"全部档位"。
- `包含补全模型`：是否同时下 LaMa Inpainting（额外 ~200MB）。

### 10.4 推荐工作流

**简单产品图分层**：

```text
[Tikpan: GPT-Image-2 官方生图] -> [Tikpan: 智能分层 PSD 生成器（经济档）]
                                        ↓
                             output/产品_xxx.psd（背景层 + 产品层）
```

**复杂海报图分层**（含主体、产品、标题、副文案）：

```text
[Tikpan: Gemini 14图极限生图] -> [Tikpan: 智能分层 PSD 生成器（标准档，开启文字检测）]
                                        ↓
                  output/海报_xxx.psd（背景 + 主体 + N产品 + N文字层）
```

**商业级分层（被遮挡也要完整）**：

```text
[Tikpan: 任意生图节点] -> [Tikpan: 智能分层 PSD 生成器（极致档）]
                                ↓
                    output/商业_xxx.psd（每一层都是完整图，不带洞）
```

**第一次部署到新环境**（推荐先预下载）：

```text
[Tikpan: PSD 模型预下载器（标准档）] -> 等几分钟下完
                ↓
[后续生图工作流] -> [智能分层节点] -> 秒级响应
```

### 10.5 常见失败处理

- **"缺少依赖: psd-tools / opencv-python / paddleocr / sam2 ..."**：把节点的"自动安装依赖"设为"是"重新运行，或单独使用"PSD 模型预下载器"节点先下载。
- **"SAM2 不可用，降级到 rembg"**：通常是 SAM2 安装失败或显存不足。经济档使用 rembg，标准档/极致档会自动降级到 rembg 保证可用性。
- **分层效果不理想**：
  - 简单图分得太碎 → 调高 `最小元素面积`（5000-10000）。
  - 复杂图分得太粗 → 调低 `最小元素面积`（500-1000），或确保使用标准档/极致档（SAM2 精度更高）。
  - 文字没单独成层 → 检查 `检测文字` 是否开启；OCR 对装饰艺术字识别率较低。
- **被遮挡区域有空洞**：勾选 `补全被遮挡区域` 或换成极致档。
- **SAM2 模型下载慢**：SAM2 模型从 Facebook CDN 下载，如果网络不稳定可以使用代理，或手动从 https://github.com/facebookresearch/sam2/releases 下载 `sam2.1_hiera_small.pt` 放到 `ComfyUI/models/sam2/` 目录。

```text
[LoadImage: 首帧] -> [首帧图片]

```

提示词建议写动作：

```text
镜头缓慢推进，产品周围有轻微水汽流动，背景灯光柔和变化，整体保持真实摄影风格，不改变产品结构。
```

## 12. 商业化网站对接思路

ComfyUI 节点本质上是在把用户填写的参数打包成请求，发送到 Tikpan 或上游模型服务，然后把返回结果转成 ComfyUI 可用的 `IMAGE`、`VIDEO`、`AUDIO` 或文本。

如果后续你做 Web 网站，可以按这个方式映射：

| ComfyUI 概念 | 网站后台概念 |
| --- | --- |
| 节点 | 一个模型工具或一个 API endpoint |
| 节点参数 | 前端表单字段 |
| 工作流 | 后端任务编排 |
| `api_key` | 后台供应商渠道密钥，不暴露给用户 |
| `task_id` | 数据库任务记录 |
| `VIDEO` / `AUDIO` / `IMAGE` 输出 | 用户作品文件、预览地址、下载地址 |
| 日志字符串 | 后台任务日志和错误详情 |
| `max_wait_seconds` / `poll_interval` | 队列轮询和超时策略 |

网站化时不要让用户填写供应商密钥。用户只需要选择模型、上传素材、填写提示词、确认价格并提交。后台根据模型、分辨率、时长、张数等参数计算费用，然后选择你配置的供应商渠道提交任务。

## 13. 常见问题排查

### 12.1 提示 API Key 无效

检查：

1. 密钥是否来自 https://tikpan.com。
2. 是否填到了当前节点的密钥字段。
3. 是否多复制了空格、换行。
4. 账号余额或权限是否正常。

### 12.2 视频任务超时

处理方式：

1. 把 `mode` 改成异步。
2. 保存返回的 `task_id`。
3. 使用 `Tikpan: 异步任务查询与下载` 查询。
4. 适当调大 `max_wait_seconds`，查询间隔保持 5-10 秒。

### 12.3 上游已经扣费，但本地没下载到文件

优先看节点日志中的：

```text
task_id
cloud_url
local_path
error message
```

如果有任务 ID，用 Task Fetcher 继续查询。若有云端链接，可以先用浏览器打开确认资源是否生成成功。

### 12.4 `VIDEO` 或 `AUDIO` 不能预览

原因通常是当前 ComfyUI 环境缺少对应预览节点或视频/音频扩展。优先看字符串输出中的本地路径和云端链接，文件本身仍可能已经生成。

### 12.5 图片太大或上传失败

处理方式：

1. 先压缩参考图。
2. 降低分辨率测试。
3. 能用 URL 就优先用 URL。
4. 减少参考图数量。

### 12.6 429 请求过多

处理方式：

1. 降低并发。
2. 增大任务间隔。
3. 等待 30-60 秒后重试。
4. 商业网站后端要做队列限流。

### 12.7 余额不足或额度不足

检查 Tikpan 账户余额。后续做网站时，也要在用户提交前完成预估扣费，避免任务提交后余额不足。

## 14. 推荐使用习惯

1. 先用低分辨率和短时长测试提示词。
2. 满意后再切到高分辨率、高画质或更长视频。
3. 视频任务尽量用异步模式。
4. 每次任务都保存 `task_id`、云端链接、本地路径和日志。
5. 商业批量任务不要只依赖 ComfyUI 前台等待，应该用后台队列轮询。
6. 多参考图时，在提示词里明确每张图的用途，例如“图1保持人物，图2参考服装，图3参考背景”。
7. 修图时明确“必须保留”的元素，比如 logo、文字、人物脸部、产品比例。

## 15. 错误码参考

| 错误 | 含义 | 处理 |
| --- | --- | --- |
| `401` | API Key 无效 | 检查密钥、账号、复制格式 |
| `402` / `insufficient_quota` | 余额或额度不足 | 充值或切换可用渠道 |
| `403` | 无权限 | 检查账号权限和模型权限 |
| `429` | 请求过多 | 降低并发，稍后重试 |
| `500` | 上游服务错误 | 稍后重试，保留日志 |
| `502` / `503` / `504` | 服务不可用或超时 | 使用异步模式或 Task Fetcher |
| `invalid_request_error` | 参数错误 | 检查提示词、比例、尺寸、输入文件 |
| `unknown_parameter` | 参数不兼容 | 更新节点或对照最新参数 |
