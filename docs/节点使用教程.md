# Tikpan Pro ComfyUI 节点使用教程

> 更新日期：2026-05-11  
> 适用项目：ComfyUI-Tikpan-Pro  
> 节点分类：`👑 Tikpan 官方独家节点`  
> 官方入口：https://tikpan.com

这份文档的目标不是堆参数，而是让使用者能快速判断“该用哪个节点、怎么连接、结果从哪里拿、失败了怎么处理”。如果你后续要把这些能力搬到自己的 Web 网站里，也可以把这里的节点参数当成一套已经验证过的请求模板。

---

## 1. 快速开始

### 1.1 安装位置

把整个插件目录放到 ComfyUI 的自定义节点目录：

```text
ComfyUI/custom_nodes/ComfyUI-Tikpan-Pro
```

然后重启 ComfyUI。右键画布添加节点时，找到：

```text
👑 Tikpan 官方独家节点
```

### 1.2 获取 API Key

1. 打开 https://tikpan.com 并登录账号。
2. 进入控制台或个人中心，创建 API Key。
3. 在节点的 `api_key`、`API_密钥` 或 `Tikpan_API密钥` 字段中填入密钥。

不同节点的密钥字段名称略有不同，但含义一致：都是 Tikpan 的接口调用凭证。

### 1.3 最小可用工作流

文生图：

```text
[Tikpan: GPT-Image-2 官方正式版(生图)] -> [PreviewImage]
```

图生图或参考图生成：

```text
[LoadImage] -> [参考图/主图输入]
[Tikpan: Gemini 14图极限生图] -> [PreviewImage]
```

文生视频：

```text
[Tikpan: HappyHorse 1.0 T2V 文生视频] -> [PreviewVideo 或视频保存节点]
```

音乐生成：

```text
[Tikpan: Suno 音乐生成] -> [PreviewAudio 或 SaveAudio]
```

语音合成：

```text
[Tikpan: speech-2.8-hd 高清语音合成] -> [PreviewAudio 或 SaveAudio]
[Tikpan: speech-2.8-turbo 极速语音合成] -> [PreviewAudio 或 SaveAudio]
[Tikpan: Gemini 3.1 Flash TTS Preview] -> [PreviewAudio 或 SaveAudio]
```

## 3. 通用参数怎么理解

| 参数 | 含义 | 建议 |
| --- | --- | --- |
| `api_key` / `API_密钥` / `Tikpan_API密钥` | Tikpan 调用凭证 | 不要截图公开，不要提交到公开仓库 |
| `prompt` / `生成指令` / `修改指令` | 你要模型完成的任务 | 写清主体、场景、风格、构图、限制 |
| `negative_prompt` / `负面风格标签` | 不希望出现的内容 | 用于排除低质量、错误风格或不需要的元素 |
| `seed` | 随机种子 | 固定 seed 方便复现；随机 seed 方便探索 |
| `resolution` / `分辨率档位` | 输出清晰度 | 草稿用低档，最终图再升高 |
| `ratio` / `aspect_ratio` / `画面比例` | 横竖屏比例 | TikTok/短视频常用 `9:16`，横版常用 `16:9` |
| `duration` | 视频时长 | 时长越长越慢，费用通常也更高 |
| `watermark` | 是否带水印 | 商用素材通常关闭水印 |
| `mode` | 同步或异步提交 | 长视频建议异步，再用 Task Fetcher 查询 |
| `max_wait_seconds` | 最长等待时间 | 同步等待时使用；过短会提前超时 |
| `poll_interval` | 查询间隔 | 一般 5-10 秒即可，不建议太频繁 |
| `代理端口` | 本地代理端口 | 只有部分旧链路需要，常见为 `10808` 或 `7890` |

## 4. 输出怎么连接

### 4.1 图片输出

图片节点一般输出：

```text
IMAGE
STRING 日志
```

连接方式：

```text
[Tikpan 图片节点: IMAGE] -> [PreviewImage]
[Tikpan 图片节点: IMAGE] -> [SaveImage]
```

日志字符串可以接到文本显示节点，也可以直接看 ComfyUI 控制台。日志里通常包含请求参数、下载路径、错误信息和上游返回摘要。

### 4.2 视频输出

视频节点一般输出：

```text
本地保存路径
任务ID
云端链接
完整日志
VIDEO
```

推荐同时保留 `任务ID` 和 `云端链接`。如果 ComfyUI 当前环境没有视频预览节点，`VIDEO` 不能直接看也没关系，本地保存路径和云端链接仍然可用。

### 4.3 音频输出

Suno 节点会输出：

```text
音频路径1 / 音频路径2
音频链接1 / 音频链接2
任务ID
片段ID
AUDIO 1
AUDIO 2
```

连接方式：

```text
[Tikpan: Suno 音乐生成: AUDIO] -> [PreviewAudio]
[Tikpan: Suno 音乐生成: AUDIO] -> [SaveAudio]
```

speech-2.8-hd / speech-2.8-turbo 节点会输出单路语音：

```text
音频路径
音频链接
任务ID
文件ID
计费字符数
状态日志
AUDIO
```

连接方式：

```text
[Tikpan: speech-2.8-hd 高清语音合成: AUDIO] -> [PreviewAudio]
[Tikpan: speech-2.8-turbo 极速语音合成: AUDIO] -> [PreviewAudio]
```

Gemini TTS 节点会输出单路 WAV 音频，并额外输出接口路径和 token 用量：

```text
音频路径
音频链接
输入输出用量
接口路径
状态日志
AUDIO
```

连接方式：

```text
[Tikpan: Gemini 3.1 Flash TTS Preview: AUDIO] -> [PreviewAudio]
[Tikpan: Gemini 3.1 Flash TTS Preview: AUDIO] -> [SaveAudio]
```

## 5. 图片节点使用方法

### 5.1 GPT-Image-2 官方正式版(生图)

推荐用于稳定的商业图像生成。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | 填 Tikpan API Key |
| `生成指令` | 写完整画面要求 |
| `模型` | 默认 `gpt-image-2` |
| `分辨率档位` | `512`、`1K`、`2K`、`4K` 等 |
| `画面比例` | `1:1`、`16:9`、`9:16`、`4:3`、`3:4`、`21:9`、`9:21` |
| `画质与推理强度` | `auto`、`low`、`medium`、`high` |
| `审核强度` | `auto`、`low`、`high` |
| `返回格式` | `png`、`webp`、`jpg` |
| `跳过错误` | 批量工作流建议开启，失败时不中断整个流程 |

推荐工作流：

```text
[Tikpan: GPT-Image-2 官方正式版(生图)] -> [PreviewImage]
                                             -> [SaveImage]
```

提示词示例：

```text
一张竖屏 9:16 的商业产品海报，主体是一瓶透明香水，放在潮湿的黑色岩石上，背景是柔和的晨光和浅雾，真实摄影风格，高级杂志广告质感，清晰产品标签，画面干净，不要多余文字。
```

### 5.2 GPT-Image-2 官方正式版(修图) V2

推荐用于编辑已有图片，支持主图、参考图、遮罩、生成多张。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `主图像` | 必填，要编辑的原图 |
| `编辑指令` | 说明要修改什么、保留什么 |
| `生成张数` | 一次返回多张变体 |
| `分辨率档位` | `Auto` 会尽量保留原图尺寸逻辑 |
| `画面比例` | `Auto` 会沿用原图比例 |
| `背景模式` | `auto`、`opaque`、`transparent` |
| `遮罩反相` | 遮罩方向不对时切换 |
| `提示增强` | 建议开启，提高稳定性 |
| `参考图1-4` | 放产品、人物、风格参考 |
| `参考流` | 可接上一节点的图片批次 |
| `遮罩掩码` | 精准控制编辑区域 |

无遮罩修图：

```text
[LoadImage: 原图] -> [主图像]
[Tikpan: GPT-Image-2 官方正式版(修图) V2] -> [PreviewImage]
```

遮罩修图：

```text
[LoadImage: 原图] -> [主图像]
[MaskEditor/遮罩节点] -> [遮罩掩码]
[Tikpan: GPT-Image-2 官方正式版(修图) V2] -> [PreviewImage]
```

遮罩规则通常是：白色区域表示要修改，黑色区域表示保留。如果结果刚好反了，打开 `遮罩反相`。

### 5.3 GPT-Image-2-all 视觉建筑师(生成)

适合多参考图生成，最多可接 14 张参考图。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | Tikpan API Key |
| `生成指令` | 说明如何使用参考图 |
| `分辨率档位` | `512`、`1K`、`2K`、`4K` |
| `画面比例` | 常用 `1:1`、`9:16`、`16:9` |
| `品质` | `standard` 更快，`hd` 更精细 |
| `参考图_1~14` | 放人物、产品、风格、场景参考 |

工作流：

```text
[LoadImage: 产品] -> [参考图_1]
[LoadImage: 风格] -> [参考图_2]
[Tikpan: GPT-Image-2-all 视觉建筑师(生成)] -> [PreviewImage]
```

### 5.4 GPT-Image-2-all 视觉整形师(修改)

适合在已有底图上做局部重绘、换产品、换背景。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `底图` | 必填，要修改的图片 |
| `修改指令` | 写清修改区域和保留区域 |
| `遮罩_Mask` | 可选，白色通常代表修改区域 |
| `产品参考图` | 可选，要植入或保持一致的产品 |
| `品质` | `standard` 或 `hd` |

工作流：

```text
[LoadImage: 底图] -> [底图]
[LoadImage: 产品] -> [产品参考图]
[MaskEditor] -> [遮罩_Mask]
[Tikpan: GPT-Image-2-all 视觉整形师(修改)] -> [PreviewImage]
```

### 5.5 Gemini 14图极限生图

适合多图融合、角色一致性、风格迁移，最多可接 14 张参考图。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | Tikpan API Key |
| `修改指令` | 既可文生图，也可要求参考图融合 |
| `模型` | 根据节点下拉选择 |
| `分辨率` | `1K`、`2K`、`4K` 等 |
| `画面比例` | 选择目标比例 |
| `调用方式` | 按节点提供的链路选择 |
| `参考图_1~14` | 多图输入 |

提示词建议：

```text
请参考图1的人物脸部特征，参考图2的服装材质，参考图3的拍摄光线，生成一张竖屏商业短视频封面。保持人物身份一致，背景更换为高端商场橱窗，真实摄影风格。
```

### 5.6 Nano Banana Pro

适合 Gemini / Banana Pro 图像链路，支持多参考图、温度和 `max_tokens` 控制。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `调用方式` | 在节点中选择 Gemini 原生或兼容调用方式 |
| `模型` | 如 `gemini-3-pro-image-preview` 等 |
| `修改指令` | 生成或编辑要求 |

| `温度` | 越高越发散，越低越稳定 |
| `max_tokens` | 控制响应文本/多模态预算 |

### 5.7 豆包图像生成

常用字段：

| 字段 | 用法 |
| --- | --- |

| `prompt` | 生成提示词 |
| `size_mode` / `size_quality` | 控制尺寸和质量 |
| `aspect_ratio` | 画面比例 |
| `output_format` | 输出格式 |
| `watermark` | 是否加水印 |

| `多图生成` | 一次生成多张 |
| `最多生成张数` | 控制批量数量 |
| `input_image` | 可选参考图 |
| `image_url_or_base64` | 可传图片 URL 或 base64 |
| `negative_prompt` | 可选负面提示词 |

## 6. 视频节点使用方法

### 6.1 同步模式和异步模式

HappyHorse 系列节点有 `mode` 参数：

| 模式 | 行为 | 适合场景 |
| --- | --- | --- |
| 同步 | 节点提交任务后持续等待，直到拿到视频或超时 | 小任务、测试任务 |
| 异步 | 节点提交后尽快返回任务 ID，后续用 Task Fetcher 查询 | 长任务、批量任务、商业工作流 |

视频生成经常耗时较长。建议正式工作流优先使用异步模式，因为它更适合排队、重试和后续网站化。

### 6.2 HappyHorse T2V 文生视频

常用字段：

| 字段 | 用法 |
| --- | --- |
| `api_key` | Tikpan API Key |
| `prompt` | 视频提示词 |
| `mode` | 同步或异步 |
| `resolution` | 输出分辨率 |
| `ratio` | 画面比例 |
| `duration` | 视频时长 |
| `watermark` | 是否水印 |
| `seed` | 随机种子 |
| `max_wait_seconds` | 同步最长等待 |
| `poll_interval` | 查询间隔 |

工作流：

```text
[Tikpan: HappyHorse 1.0 T2V 文生视频] -> [PreviewVideo]
                                          -> [保存/后处理视频节点]
```

### 6.3 HappyHorse I2V 图生视频

适合从一张首帧图开始生成视频。

工作流：

```text
[LoadImage: 首帧] -> [首帧图片]
[Tikpan: HappyHorse 1.0 I2V 图生视频] -> [PreviewVideo]
```

如果图片已经在云端，也可以使用 `图片URL`，这样通常比本地上传更稳定。

### 6.4 HappyHorse R2V 参考生视频

适合多参考图视频。最多可输入多张参考图，也可填写图片 URL 列表。

工作流：

```text
[LoadImage: 人物] -> [参考图1]
[LoadImage: 产品] -> [参考图2]
[LoadImage: 场景] -> [参考图3]
[Tikpan: HappyHorse 1.0 R2V 参考生视频] -> [PreviewVideo]
```

### 6.5 HappyHorse Video-Edit 视频编辑

适合修改已有视频，可传 `视频URL` 或 `本地视频`，并支持参考图。

工作流：

```text
[LoadVideo/视频输入] -> [本地视频]
[LoadImage: 产品参考] -> [参考图1]
[Tikpan: HappyHorse 1.0 Video-Edit 视频编辑] -> [PreviewVideo]
```

### 6.6 Grok-Videos 视频生成

常用字段：

| 字段 | 用法 |
| --- | --- |
| `api_key` | Tikpan API Key |
| `prompt` | 视频提示词 |
| `model` | 节点下拉模型 |
| `duration` | 时长 |
| `aspect_ratio` | 画面比例 |
| `ref_image_1~4` | 可选参考图 |

输出包含本地路径、云端直链、日志和 `VIDEO`。

### 6.7 Veo 3.1 多模型视频生成

适合需要 Google Veo 3.1 链路时使用。当前节点只保留以下 7 个 Tikpan 价格页中展示的模型，并按模型分别走对应端点。

```text
POST https://tikpan.com/v1/videos
POST https://tikpan.com/v1/video/create
```

核心参数保持一致：`model`、`prompt`、`aspect_ratio`、`seed`。根据官方 Veo 3.1 文档，比例使用 `16:9` 或 `9:16`；components 类模型支持最多 3 张参考垫图。`/v1/videos` 使用 OpenAI 视频格式表单提交；`/v1/video/create` 使用视频统一 JSON 格式提交。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | Tikpan API Key |
| `Veo专属提示词` | 视频生成提示词 |
| `模型选择` | 选择 lite、fast、components 或 4K 模型 |
| `比例` | `16:9` 或 `9:16` |
| `seed` | 随机种子 |
| `参考图_首帧` | 可选，用一张图控制视频第一帧 |
| `参考图_尾帧` | 可选，用一张图控制视频结束帧 |
| `垫图_1~3` | 可选，仅 components 模型使用，最多 3 张参考垫图 |

当前模型选项：

| 模型 | 建议用途 |
| --- | --- |
| `veo_3_1-lite` | `/v1/videos`，低成本 lite 模式，适合日常测试 |
| `veo_3_1-lite-4K` | `/v1/videos`，lite 4K，高性价比 |
| `veo_3_1-fast-4K` | `/v1/videos`，快速 4K，支持首尾帧和文生视频 |
| `veo3.1-fast-components` | `/v1/video/create`，更快、更稳、更可控，支持最多 3 张垫图 |
| `veo3.1-pro` | `/v1/video/create`，高质量 pro 模式，价格较高 |

### 7.2 speech-2.8-hd / speech-2.8-turbo 语音合成

适合短视频旁白、广告口播、产品解说、多语言配音。`speech-2.8-hd` 更偏高质量细节，`speech-2.8-turbo` 更偏快速响应和批量预览。默认走同步接口，长文本建议切换为异步接口。

常用字段：

| 字段 | 用法 |
| --- | --- |
| `API_密钥` | Tikpan API Key |
| `合成文本` | 需要朗读的文本；可使用 `<#0.5#>` 控制停顿 |
| `调用方式` | 短文本用同步，长文本用异步 |
| `音色ID` | 系统音色、复刻音色或音色设计生成的 voice_id |
| `语言增强` | 不确定语言时用 `auto`，明确中文/英文时可手动选择 |
| `语速` / `音量` / `音调` | 常用声音控制参数 |
| `情绪` | 不需要时保持“默认不传” |
| `音频格式` | `mp3` 兼容性最好，`wav/flac` 适合后期处理 |
| `复用本地缓存` | 建议开启；相同参数命中缓存时不会重新请求上游 |

工作流：

```text
[Tikpan: speech-2.8-hd 高清语音合成] -> [PreviewAudio]
                                      -> [SaveAudio]
[Tikpan: speech-2.8-turbo 极速语音合成] -> [PreviewAudio]
                                         -> [SaveAudio]
```

商业化注意事项：

- `计费字符数` 输出可直接用于网站侧的用量统计和余额流水。
- 提交后网络断开时，不要马上改参数重跑；先看 `recovery/speech_2_8_hd` 里的恢复记录，避免重复扣费。
- 自定义音色和快速复刻音色可能产生额外费用，网站端应把 voice_id 来源、创建费用和合成费用分开记录。

适合低延迟、可控语音生成。节点默认走 Gemini 原生 `generateContent`，按官方 TTS 结构返回音频；如果上游返回裸 PCM，节点会自动封装成 WAV。

| `合成文本` | 需要朗读的文本 |
| `调用方式` | `geminitts 原生`、`gemini 原生` 或 `openai 兼容` |

| `语言代码` | 可选，如 `zh-CN`、`en-US` |
| `复用本地缓存` | 建议开启；相同参数命中缓存时不会重新请求上游 |

工作流：


商业化注意事项：

## 10. 商业化网站对接思路

ComfyUI 节点本质上是在把用户填写的参数打包成请求，发送到 Tikpan 或上游模型服务，然后把返回结果转成 ComfyUI 可用的 `IMAGE`、`VIDEO`、`AUDIO` 或文本。

如果后续你做 Web 网站，可以按这个方式映射：

| ComfyUI 概念 | 网站后台概念 |
| --- | --- |
| 节点 | 一个模型工具或一个 API endpoint |
| 节点参数 | 前端表单字段 |
| 工作流 | 后端任务编排 |
| `api_key` | 后台供应商渠道密钥，不暴露给用户 |
| `task_id` | 数据库任务记录 |
| `VIDEO` / `AUDIO` / `IMAGE` 输出 | 用户作品文件、预览地址、下载地址 |
| 日志字符串 | 后台任务日志和错误详情 |
| `max_wait_seconds` / `poll_interval` | 队列轮询和超时策略 |

网站化时不要让用户填写供应商密钥。用户只需要选择模型、上传素材、填写提示词、确认价格并提交。后台根据模型、分辨率、时长、张数等参数计算费用，然后选择你配置的供应商渠道提交任务。

## 11. 常见问题排查

### 11.1 提示 API Key 无效

检查：

1. 密钥是否来自 https://tikpan.com。
2. 是否填到了当前节点的密钥字段。
3. 是否多复制了空格、换行。
4. 账号余额或权限是否正常。

### 11.2 视频任务超时

处理方式：

1. 把 `mode` 改成异步。
2. 保存返回的 `task_id`。
3. 使用 `Tikpan: 异步任务查询与下载` 查询。
4. 适当调大 `max_wait_seconds`，查询间隔保持 5-10 秒。

### 11.3 上游已经扣费，但本地没下载到文件

优先看节点日志中的：

```text
task_id
cloud_url
local_path
error message
```

如果有任务 ID，用 Task Fetcher 继续查询。若有云端链接，可以先用浏览器打开确认资源是否生成成功。

### 11.4 `VIDEO` 或 `AUDIO` 不能预览

原因通常是当前 ComfyUI 环境缺少对应预览节点或视频/音频扩展。优先看字符串输出中的本地路径和云端链接，文件本身仍可能已经生成。

### 11.5 图片太大或上传失败

处理方式：

1. 先压缩参考图。
2. 降低分辨率测试。
3. 能用 URL 就优先用 URL。
4. 减少参考图数量。

### 11.6 429 请求过多

处理方式：

1. 降低并发。
2. 增大任务间隔。
3. 等待 30-60 秒后重试。
4. 商业网站后端要做队列限流。

### 11.7 余额不足或额度不足

检查 Tikpan 账户余额。后续做网站时，也要在用户提交前完成预估扣费，避免任务提交后余额不足。

## 12. 推荐使用习惯

1. 先用低分辨率和短时长测试提示词。
2. 满意后再切到高分辨率、高画质或更长视频。
3. 视频任务尽量用异步模式。
4. 每次任务都保存 `task_id`、云端链接、本地路径和日志。
5. 商业批量任务不要只依赖 ComfyUI 前台等待，应该用后台队列轮询。
6. 多参考图时，在提示词里明确每张图的用途，例如“图1保持人物，图2参考服装，图3参考背景”。
7. 修图时明确“必须保留”的元素，比如 logo、文字、人物脸部、产品比例。

## 13. 错误码参考

| 错误 | 含义 | 处理 |
| --- | --- | --- |
| `401` | API Key 无效 | 检查密钥、账号、复制格式 |
| `402` / `insufficient_quota` | 余额或额度不足 | 充值或切换可用渠道 |
| `403` | 无权限 | 检查账号权限和模型权限 |
| `429` | 请求过多 | 降低并发，稍后重试 |
| `500` | 上游服务错误 | 稍后重试，保留日志 |
| `502` / `503` / `504` | 服务不可用或超时 | 使用异步模式或 Task Fetcher |
| `invalid_request_error` | 参数错误 | 检查提示词、比例、尺寸、输入文件 |
| `unknown_parameter` | 参数不兼容 | 更新节点或对照最新参数 |
