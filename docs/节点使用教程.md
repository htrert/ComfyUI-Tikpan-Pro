## 1️⃣ Tikpan：Gemini 14图极限生图

### 节点说明
**图标**：👑  
**功能**：使用Google Gemini模型进行图生图，支持最多14张参考图  
**适用场景**：极限分辨率创作、多图融合、风格迁移

### 参数详解

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| API_密钥 | STRING | sk- | Tikpan API密钥 |
| 修改指令 | STRING | 请结合参考图... | 生成指令，中英文均可 |
| 模型 | SELECT | gemini-3.1-flash-image-preview | 可选：gemini-2.0-flash-exp |
| **分辨率** | SELECT | 1K | 512/1K/2K/4K + 画面比例动态计算 |
| **画面比例** | SELECT | 9:16 | 1:1/16:9/9:16/21:9/4:3 |
| seed | INT | 888888 | 随机种子 |
| 生成后控制 | SELECT | randomize | fixed/randomize/increment/decrement |
| 参考图_1~14 | IMAGE | 可选 | 最多14张参考图 |

### 分辨率计算示例

| 分辨率档位 | 画面比例 | 实际输出尺寸 |
|-----------|---------|-------------|
| 4K | 9:16 | 4096×7280 |
| 2K | 16:9 | 2048×1152 |
| 1K | 9:16 | 1024×1816 |
| 512 | 21:9 | 512×208 |

### 使用技巧

1. **多图融合**：连接多张参考图，Gemini会自动融合风格
2. **保持seed**：固定seed可复现相似结果
3. **分辨率选择**：
   - 512：快速预览/测试
   - 1K：日常创作
   - 2K：高精度输出
   - 4K：极限高清（文件较大，生成较慢）

### 工作流示例

```
[LoadImage] → [LoadImage] → [LoadImage] → [TikpanGeminiImage] → [PreviewImage]
     ↓            ↓            ↓
  参考图_1     参考图_2     参考图_3
```

## 2️⃣ Tikpan：GPT-Image-2 视觉建筑师(生成)

### 节点说明
**图标**：🎨  
**功能**：使用GPT-Image-2模型进行图生图，支持14路参考图  
**适用场景**：需要参考多张图片进行创作的场景

### 参数详解

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| API_密钥 | STRING | sk- | Tikpan API密钥 |
| 生成指令 | STRING | 请参考Image_1... | 详细描述生成需求 |
| 模型 | SELECT | gpt-image-2-all | 目前仅此选项 |
| 分辨率档位 | SELECT | 1K | 512/1K/2K/4K |
| 画面比例 | SELECT | 1:1 | 1:1/16:9/9:16/21:9/4:3/3:4 |
| 品质 | SELECT | hd | standard/hd |
| seed | INT | 888888 | 随机种子 |
| 代理端口 | STRING | 10808 | 本地代理端口 |
| 参考图_1~14 | IMAGE | 可选 | 参考图输入 |

### 尺寸计算逻辑

```
基础尺寸（以长边为基准）:
- 512 → 512px
- 1K  → 1024px
- 2K  → 2048px
- 4K  → 4096px

根据画面比例缩放短边，并对齐到8的倍数
```

### 使用技巧

1. **参考图使用**：
   - 缩略图预处理器（最大1024px）
   - JPEG质量75%压缩
   - 最多14张参考图
2. **品质选择**：
   - `standard`：速度快，消耗低
   - `hd`：细节丰富，消耗高

## 3️⃣ Tikpan：GPT-Image-2 视觉整形师(修改)

### 节点说明
**图标**：💉  
**功能**：对已有图片进行局部修改  
**适用场景**：产品换背景、人像换装、局部重绘

### 参数详解

| 参数 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| API_密钥 | STRING | sk- | Tikpan API密钥 |
| **底图** | IMAGE | 必填 | 需要修改的原图 |
| 修改指令 | STRING | 请把图中人物... | 修改要求描述 |
| 模型 | SELECT | gpt-image-2-all | 目前仅此选项 |
| 品质 | SELECT | hd | standard/hd |
| 代理端口 | STRING | 10808 | 本地代理端口 |
| 遮罩_Mask | MASK | 可选 | 白色=修改区域，黑色=保留区域 |
| 产品参考图 | IMAGE | 可选 | 要放入图中的产品图 |

### 遮罩使用指南

1. **遮罩规范**：
   - PNG格式带Alpha通道
   - 白色/浅色 = 要修改的区域
   - 黑色/深色 = 保留的区域
2. **尺寸要求**：
   - 遮罩会自动缩放到与底图一致
   - 自动对齐到8的倍数

### 工作流示例

```
[LoadImage] → [Mask] → [TikpanEdit] → [PreviewImage]
    ↓          ↓
  底图        遮罩
```

### 使用技巧

1. **无遮罩时**：AI会自行判断需要修改的区域
2. **有产品参考图时**：产品会被融入底图指定位置
3. **保持一致性**：提示词中强调保持光线和视角一致效果更好

## ❓ 常见问题

### Q1: 生成失败，提示"API密钥无效"
**解决**：
1. 检查密钥是否以`sk-`开头
2. 确认密钥是否在tikpan.com创建
3. 检查密钥是否已过期或额度用尽

### Q2: 生成速度很慢
**解决**：
1. 使用较低分辨率进行测试
2. 检查网络代理是否稳定
3. 尝试切换到更稳定的时段

### Q3: 图片分辨率不正确
**解决**：
1. 确保选择正确的分辨率档位
2. 检查画面比例是否正确设置
3. 更新节点到最新版本

### Q4: 429 错误（请求过多）
**解决**：
1. 等待30-60秒后重试
2. 减少并发请求
3. 检查API额度

### Q5: 提示词审核被拦截
**解决**：
1. 使用更中性的描述
2. 尝试降低审核强度到low
3. 避免使用敏感词汇

## 📊 错误代码解释

| 错误代码 | 含义 | 解决方案 |
|---------|------|---------|
| 401 | API密钥无效 | 检查密钥是否正确 |
| 403 | 无权限访问 | 确认账户权限 |
| 429 | 请求频率超限 | 等待后重试 |
| 500 | 服务器内部错误 | 稍后重试 |
| 502/503/504 | 服务不可用 | 联系官方支持 |

### Tikpan特有错误

| 错误信息 | 含义 | 解决方案 |
|---------|------|---------|
| insufficient_quota | 余额不足 | 充值账户 |
| unknown_parameter | 参数不兼容 | 更新节点 |
| invalid_request_error | 请求参数错误 | 检查输入格式 |
