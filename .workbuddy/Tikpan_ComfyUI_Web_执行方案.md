# Tikpan 平台新增 ComfyUI 前端 — 执行方案

> 整理自 2026-05-07 与 AI 的技术讨论
> 目标：在现有 tikpan.com 中转站基础上，新增一个 Web 前端面板，让用户通过网页输入提示词和参考图，自动调用 API 出图

---

## 一、当前架构梳理

### 1.1 你现在有什么

| 组件 | 作用 | 状态 |
|------|------|------|
| tikpan.com | 中转站（指向器） | ✅ 已上线，指向美国上游 |
| 上游算力 (api.tpkcur.xyz) | 真正跑 ComfyUI 的服务器 | ✅ 在美国 |
| tikpan.com 域名 | 在 Cloudflare 管理，灰色云朵（仅 DNS） | ✅ 正常使用 |

### 1.2 新增的目标架构

```
国内用户
    │
    ▼
香港服务器 (2核4G) ←─── 只传JSON指令（几KB）
    │              不走任何图片流量
    │
    ├──→ 阿里云香港 OSS ←─── 用户直传参考图（百兆速度）
    │
    └──→ Tikpan 中转站 ──→ 美国上游算力
                    │
                    └──→ 返回图片链接
```

**关键原则：解耦。图片流量和指令流量完全分开。**

## 二、为什么必须用香港

### 2.1 物理位置决定一切

你的上游算力服务器在美国。如果：
- **香港服务器 → 美国上游**：走国际带宽，延迟低且稳定（CN2 GIA 专线）
- **国内服务器 → 美国上游**：国际出口受限，容易丢包、超时

香港是"国内用户"和"美国算力"之间的黄金平衡点。

### 2.2 国内用户体验

配合阿里云国内 CDN 后：
- 用户下载生成图片：从同城的阿里云节点拉取 = **秒开**
- 用户上传参考图：直传到香港 OSS = **瞬间完成**

### 2.3 Cloudflare 小黄云 —— 绝对不能开

| 状态 | 效果 |
|------|------|
| 灰色云朵（仅DNS）| tikpan.com → 直连香港服务器 IP，延迟 30-50ms ✅ |
| 小黄云（开启代理）| 国内请求先绕道 Cloudflare 美国节点，再回香港，延迟飙到 200-300ms ❌ |

**结论：你的域名 tikpan.com 和任何新增的二级域名，一律保持灰色云朵。**

## 三、服务器配置定论

### 3.1 推荐配置

| 项目 | 推荐配置 | 理由 |
|------|----------|------|
| **位置** | 中国香港 | 桥梁作用：接国内用户 + 连美国上游 |
| **线路** | CN2 GIA / BGP 多线优化 / 大陆直连 | 选机器时一定要看线路描述，普通国际线路晚高峰卡死 |
| **CPU/内存** | **2核 4G**（起步完全够用） | 2核2G也可以，但4G更稳 |
| **带宽计费** | **按流量计费**，峰值拉到 100Mbps | 固定带宽极贵，按流量只有 0.8元/GB |
| **系统盘** | 50GB SSD | 够用 |

### 3.2 为什么 2核4G 够用？

因为你的服务器只做"文本路由"：
- 用户登录/充值查询：纯文本
- 发指令给 Tikpan 中转站：几KB 的 JSON
- 用户上传/下载图片：**全部走 OSS，完全不经过服务器带宽**

内存分配推演（2核4G）：
- Linux 系统：~400MB
- Nginx/Python/Node.js：~500MB
- MySQL（用户余额数据库）：~500MB
- 剩余可用：~2.5GB 应对并发高峰

### 3.3 两个必须做的配置

1. **开启 Swap 虚拟内存**（2-4GB）：防止内存偶尔飙满时系统崩溃
2. **不装/少装宝塔插件**：宝塔面板本身常驻吃掉几百兆，全装上 2G 秒见底。只装必需：Nginx + Python/Node + MySQL。

## 四、OSS 对象存储方案

### 4.1 为什么必须用 OSS

如果图片走服务器带宽：
- 用户上传 10MB 参考图 → 你的 10M 服务器带宽直接堵死
- 10个人并发 → 服务器宕机

用 OSS 后：
- 用户浏览器直连阿里云 OSS 上传图片（百兆/千兆速度）
- 服务器只收到一个图片链接 `https://xxx.oss.com/user/abc.jpg`
- 服务器把这个链接发给 Tikpan 中转站 = 几KB 的文本指令

### 4.2 技术实现：客户端直传（STS 临时授权）

这是最安全的"绕开服务器带宽"方案：

```
步骤1: 用户点上传 → 前端向你的服务器请求"临时通行证"
步骤2: 你的服务器向 OSS 申请 STS 临时凭证（几行代码）
步骤3: 前端拿到凭证 → 浏览器直连 OSS 传图
步骤4: 传完 → 拿到 OSS 链接 → 发给你的服务器
步骤5: 你的服务器把链接 + 提示词 → 发给 Tikpan 中转站
```

⚠️ **安全要点**：AccessKey 绝对不能写在前端代码里，必须走后台 STS 临时授权。

### 4.3 推荐配置

| 项目 | 推荐 |
|------|------|
| **厂商** | 阿里云 OSS（生态最成熟，SDK 完善） |
| **地域** | **香港地域**（和美国上游同区域，拉图最快） |
| **额外服务** | 阿里云国内 CDN（让国内用户下载图片秒开） |
| **计费** | 按流量计费，比固定带宽便宜几十倍 |

## 五、域名规划（解耦方案）

### 5.1 最佳结构（让官网和代理站互不干扰）

| 域名 | 指向 | 作用 |
|------|------|------|
| tikpan.com / www.tikpan.com | **你自己的香港服务器** | 官网/业务介绍/引流页面 |
| console.tikpan.com 或 dash.tikpan.com | CNAME → api.tpkcur.xyz，发给云雾绑定 | 代理控制台 |
| 新增的二级域名（如 web.tikpan.com） | CNAME → 你的香港服务器 IP（灰色云朵） | ComfyUI 前端网站 |

### 5.2 为什么之前主域名进不去？

因为你把根域名 `tikpan.com`（不是二级域名）直接 CNAME 指向了云雾的系统。当你想把 `tikpan.com` 指向你自己的建站服务器时，DNS 冲突了。

**解决方案**：用**二级域名**（如 `web.tikpan.com`）指向你的新网站，根域名维持现状，解耦后互不干扰。

## 六、完整执行步骤

### 第一阶段：购买服务器（1小时）

1. **选择厂商**：阿里云国际站 / 腾讯云国际站
2. **产品类型**：轻量应用服务器（香港专区）
3. **必选配置**：
   - 地域：**香港**
   - 规格：**2核 4G**
   - 系统：**Ubuntu 22.04 LTS**（推荐）或 CentOS
   - 带宽：**按流量计费**，峰值 **100Mbps**，每月流量包 1-2TB
4. **付款后**：获得服务器公网 IP（如 `1.2.3.4`）

### 第二阶段：域名解析（10分钟）

1. 打开 Cloudflare
2. 新增一条 DNS 记录：
   - **类型**：A 记录
   - **名称**：`web`（即 `web.tikpan.com`）
   - **内容**：你的服务器公网 IP `1.2.3.4`
   - **代理状态**：👤 灰色云朵（仅 DNS）
3. 同理新增 `api` 记录指向 apifox.cn（如果还没有）
4. 保存

### 第三阶段：服务器基础环境（30分钟）

SSH 连接服务器后，依次执行：

```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装 Python + pip
sudo apt install python3 python3-pip -y

# 3. 安装 Nginx
sudo apt install nginx -y

# 4. 安装 MySQL（或用阿里云 RDS，更省心）
sudo apt install mysql-server -y

# 5. 配置 Swap（2GB）
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 6. 安装防火墙，只开必要端口
sudo ufw allow 22    # SSH
sudo ufw allow 80   # HTTP
sudo ufw allow 443  # HTTPS
sudo ufw enable
```

### 第四阶段：购买并配置阿里云 OSS（30分钟）

1. 登录阿里云国际站 → 对象存储 OSS → 开通服务
2. 创建 Bucket：
   - **Bucket 名称**：如 `tikpan-images`
   - **地域**：香港
   - **存储类型**：标准存储
3. 获取 AccessKey（密钥对），**只存在服务器后台**，绝不暴露给前端
4. 配置 CDN 加速：
   - 在 OSS 控制台开启 CDN
   - 获取一个国内 CDN 域名（如 `xxx.oss-cn-shanghai.aliyundocdn.com`）
5. 配置跨域（CORS）：允许你的域名请求 OSS

### 第五阶段：开发 Web 前端（核心部分）

#### 方案 A：基于现有 Web 应用扩展（推荐）

你已经有 `web_app/` 目录下的 Flask 网站，可以在此基础上扩展：

1. **新增图片上传路由**（用阿里 OSS SDK）：
   ```python
   # 后端只给前端一个 STS 临时凭证
   @app.route('/api/get-upload-token')
   def get_upload_token():
       # 向阿里云请求临时凭证，返回给前端
       return {'token': 'xxx', 'upload_url': 'https://tikpan-images.oss-cn-hongkong.aliyuncs.com'}
   ```

2. **新增 ComfyUI 生图路由**：
   ```python
   @app.route('/api/generate', methods=['POST'])
   def generate():
       data = request.json
       prompt = data['prompt']
       image_url = data['image_url']  # 用户上传后的 OSS 链接
       # 构造 ComfyUI 指令，发给 Tikpan 中转站
       # 返回生图结果链接
   ```

3. **前端上传组件**：用阿里云 OSS 的 JS SDK，浏览器直传

#### 方案 B：从零搭建（如果你想要更漂亮的 UI）

推荐技术栈：
- **前端**：Vue 3 + Vite（现代、快速）
- **后端**：Flask 或 FastAPI（轻量）
- **UI 组件库**：Element Plus / Naive UI
- **部署**：Nginx 反向代理 + Gunicorn

### 第六阶段：SSL 证书 + 部署上线（20分钟）

```bash
# 安装 Certbot（自动续期 SSL 证书）
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d web.tikpan.com

# 重载 Nginx
sudo systemctl reload nginx
```

### 第七阶段：测试（30分钟）

1. 用国内手机开流量访问 `web.tikpan.com`
2. 上传一张参考图，看是否走 OSS 链接
3. 输入提示词，点击生成，看图片返回链路
4. 用 https://ping.pe/ 测试各地区延迟

## 八、风险提示

1. **不要开 Cloudflare 小黄云** → 国内访问会绕路到美国，极慢
2. **不要把 AccessKey 写在前端** → API Key 泄露会导致额度被刷光
3. **不要装太多宝塔插件** → 2G 内存扛不住
4. **一定要开 Swap** → 小内存服务器的保命线
5. **OSS 地域选香港** → 不要选国内地域，否则美国上游拉图要走国际带宽
