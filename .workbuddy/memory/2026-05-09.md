## 完成：HappyHorse 1.0 I2V 图生视频节点

### 创建的文件
1. `nodes/tikpan_happyhorse_i2v.py` - HappyHorse I2V 节点完整实现（~460行）

### 节点功能
- **模型**：happyhorse-1.0-i2v（阿里百炼平台）
- **功能**：以首帧图片为基础，通过文本描述引导，生成物理真实、运动流畅的视频
- **API 端点**（中转站 tikpan.com）：
  - 提交任务：`/alibailian/api/v1/services/aigc/video-generation/video-synthesis`
  - 查询任务：`/alibailian/api/v1/tasks/{task_id}`

### 支持的参数
| 参数 | 类型 | 说明 |
|------|------|------|
| api_key | STRING | Tikpan 中转站 API 密钥 |
| prompt | STRING | 文本提示词（可选，描述动作） |
| resolution | ["720P", "1080P"] | 视频分辨率 |
| duration | INT (3-15) | 视频时长（秒） |
| watermark | ["无水印", "有水印"] | 是否添加水印 |
| seed | INT (-1~2147483647) | 随机种子（-1 表示随机） |
| 首帧图片 | IMAGE | ComfyUI 图像输入（可选） |
| 图片URL | STRING | 手动输入图片 URL（可选，优先于首帧图片） |

### 节点输出
- **📁_本地保存路径** - 视频下载到 ComfyUI 输出目录的本地路径
- **🆔_任务ID** - 异步任务 ID
- **🔗_视频云端链接** - 视频云端 URL（24小时有效）
- **📄_完整日志** - 完整的 API 请求/响应日志

### 实现细节
1. **图片上传**：
   - 支持 ComfyUI IMAGE 张量输入
   - 自动尝试多个上传端点获取 URL
   - 支持手动输入图片 URL（优先使用）
   - 备选方案：Base64 Data URI（可能不被 API 支持）

2. **异步任务处理**：
   - 提交任务，获取 task_id
   - 每 10 秒轮询任务状态
   - 支持最长 600 秒超时
   - 处理多种响应格式（兼容不同返回结构）

3. **视频下载**：
   - 自动下载到 ComfyUI 输出目录
   - 文件名：`HappyHorse_I2V_{task_id}.mp4`
   - 即使下载失败，也返回云端链接

### 阿里百炼 API 格式
```json
POST /alibailian/api/v1/services/aigc/video-generation/video-synthesis
Headers:
  Authorization: Bearer {api_key}
  Content-Type: application/json
  X-DashScope-Async: enable

Body:
{
  "model": "happyhorse-1.0-i2v",
  "input": {
    "prompt": "描述动作",
    "media": [
      {"type": "first_frame", "url": "图片URL"}
    ]
  },
  "parameters": {
    "resolution": "720P",
    "duration": 5,
    "watermark": false,
    "seed": 42
  }
}
```

### 节点功能
- **模型**：happyhorse-1.0-r2v（阿里百炼平台）
- **功能**：支持最多9张图片参考，精准保持创作意图，实现更强表现能力的视频生成

### 支持的参数
| 参数 | 类型 | 说明 |
|------|------|------|
| api_key | STRING | Tikpan 中转站 API 密钥 |
| prompt | STRING | 文本提示词（必填） |

      {"type": "reference", "url": "图片URL2"},
      ...
    ]
  },
  "parameters": {
    "resolution": "1080P",

  }
